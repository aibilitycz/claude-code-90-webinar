Claude Code vs Cursor

Tři rozdíly, které vás reálně ovlivní

V dubnu 2026 umí oba v zásadě to samé. Všechno ostatní (agent, skills, MCP, hooks, kontext) je prakticky stejné.

Kde AI žije Cursor = vlastní IDE. Claude Code = editor-agnostický (CLI, desktop, web, iOS, VS Code, JetBrains)
Volba modelu Cursor = Claude / GPT / Gemini. Claude Code = pouze Claude
Jak se platí Cursor = vlastní účet. Claude Code = součást Claude Pro / Max

ChatGPT je jiná kategorie — chatbot, nevidí soubory, nespouští příkazy.

Jak to chodí

  • Cumulative folders — každý krok udělá kopii předchozí složky a něco přidá. Vidíte, jak stejný agent dává jiný výsledek, když mu dáme víc kontextu.
  • Multimodal — Claude Code vidí obrázek a převezme z něj barvy, typografii, spacing.
  • Grounding — trendy si najde na webu, přidá odkaz, odkud je vzal.
  • Plan mode — u poslední úpravy (dark mode + karty) nejdřív ukáže plán. Schvalujete, pak teprve sahá na kód.
  • Výstup — HTML, které se otevírá v prohlížeči. Žádný build, žádné závislosti.

Z presenter notes

Co zmínit během dema

Kontrola výstupu

Čím důležitější výstup, tím důkladnější kontrola. Vždy: vizuál, kroky agenta, fakta, funkčnost.

Soukromí

Obsah jde do Anthropic API. Na placených plánech se nepoužívá pro trénink.

Git jako pojistka

Checkpoint = rychlý undo. Git = opravdová historie. Před větší změnou commit.

Kontextové okno

Opus 4.6: až 1 M tokenů (~750k slov). Claude Code má prompt caching — opakovaný kontext je výrazně levnější.

Z presenter notes

Co se hodí zmínit během dema

  • Permission módy: default, acceptEdits, plan, bypassPermissions. Poslední jen v kontejneru / VM.
  • Subagent může volat dalšího subagenta — vícevrstvé pipeline.
  • PreToolUse hook může zablokovat nebezpečnou akci dřív, než se stane. Některé týmy = interní DLP.
  • Worktree workflow: git worktree add + claude = izolovaná session pro rizikovou změnu.
  • claude-squad — komunitní nástroj, paralelně spouští více Claudů na jednom repu. Šetří čas u velkých refaktorů.
  • Prompt injection jako obor popsal 2022 Simon Willison (autor Djanga). Standardní téma v bezpečnosti agentů.
