Krok 2 ze 5

Nainstalujte Claude Code

Claude Code běží na víc místech — v desktopové aplikaci, v prohlížeči, v terminálu, na mobilu nebo v editoru. Pro webinář potřebujete cestu, která umí otevřít naklonovanou složku z vašeho počítače. Doporučujeme desktopovou aplikaci — je to nejjednodušší start. Pokud chcete vidět přesně to, co my během živého vysílání, zvolte terminál.

Webová aplikace claude.ai/code funguje pro běžný chat s Claude, ale neumí načíst společníka webináře — ten žije v naklonované složce na vašem disku. Proto pro webinář volte desktop nebo terminál.

  1. Otevřete claude.com/download
  2. Stáhněte instalátor pro macOS nebo Windows
  3. Nainstalujte jako jakoukoli jinou aplikaci
  4. Otevřete aplikaci a přihlaste se stejnými údaji jako u claude.com

Aplikace funguje podobně jako ChatGPT — píšete do chatu, ale Claude Code navíc umí otevřít vaše lokální složky a pracovat s nimi.

Doporučený způsob: nativní instalátor, který nevyžaduje Node.js. 

curl -fsSL https://claude.ai/install.sh | bash

V PowerShellu (ne v CMD):

irm https://claude.ai/install.ps1 | iex

Po instalaci spusťte příkaz claude v libovolné složce — při prvním spuštění se otevře prohlížeč k přihlášení.

Krok 3 ze 5

Nainstalujte Git

Git je nástroj, kterým se stahují projekty z GitHubu. Dnes ho potřebujete jen na jednu věc — stáhnout náš repozitář se společníkem webináře. Návod níže si přepněte podle toho, jaký máte operační systém.

Na macOS už možná Git máte. Otevřete Terminal (Cmd + Space → „Terminal") a zkuste:

git --version
Očekávaný výstup: něco jako git version 2.x.x. Pokud to funguje, máte hotovo.

Pokud vám macOS nabídne instalaci Xcode Command Line Tools, potvrďte — stáhne se všechno, co potřebujete (pár minut).

  1. Otevřete git-scm.com/download/win
  2. Stáhněte a nainstalujte 64-bit Git for Windows
  3. Během instalace nechte výchozí nastavení (stačí klikat „Next")
  4. Po instalaci otevřete PowerShell a ověřte:
git --version
Očekávaný výstup: něco jako git version 2.x.x.windows.x.

Podle distribuce:

# Debian, Ubuntu
sudo apt install git

# Fedora, RHEL
sudo dnf install git

# Arch
sudo pacman -S git
Ověření: git --version by mělo vrátit git version 2.x.x.

Stáhněte materiály k webináři

Teď si naklonujete náš repozitář — dostanete tím společníka webináře (skill, který vám během živého vysílání pomáhá), tři úrovně materiálů a cheat sheet pro první půlhodinu po webináři.

Otevřete terminál (macOS/Linux) nebo PowerShell (Windows) a spusťte:

git clone https://github.com/aibilitycz/claude-code-90-webinar.git
cd claude-code-90-webinar

Krok 5 ze 5

Preflight check

Posledních pár vteřin. Ověřte si, že všechno funguje — ať nás zítra ráno nic nepřekvapí.

1. Git funguje

git --version
✅ Chceme vidět: git version 2.x.x (jakákoli verze 2 je OK).

2. Jste ve správné složce

ls .claude/skills
✅ Chceme vidět: claude-code-guide — společník sedí, kde má. Pokud vám ls nefunguje (Windows PowerShell), zkuste dir .claude\skills.

3. Claude Code se ohlásí

Pokud jste zvolili terminál:

claude --version
✅ Chceme vidět: nějaké číslo verze (např. 1.x.x). Pokud ano, spusťte claude v téhle složce a zeptejte se: „Co mám zkusit poprvé?" Společník by vám měl nabídnout tři možnosti.

Pokud používáte desktopovou aplikaci:

Otevřete aplikaci, v levém menu zvolte „Open folder" a vyberte složku claude-code-90-webinar, kterou jste si právě naklonovali. V chatu se zeptejte „Co mám zkusit poprvé?". Společník by se měl ozvat s nabídkou tří možností.

Čtyři nejčastější věci, na kterých se účastníci minulých webinářů zasekli. Pokud vás nic z toho nezachrání, napište do Circle komunity — odkaz máte v registračním e-mailu.

git --version hlásí „command not found"

Git není nainstalovaný. Vraťte se ke Kroku 3 a postupujte podle návodu pro váš operační systém. Po instalaci zavřete terminál a otevřete ho znovu — stará session nový příkaz neuvidí.

claude po instalaci přes terminál hlásí „command not found"

Instalace pravděpodobně proběhla, ale binárka není na PATH. Zkuste:

1. Zavřete terminál a otevřete ho znovu (nejčastější řešení).
2. Ověřte umístění: which claude (macOS/Linux) nebo where claude (Windows).
3. Pokud dál nic, stáhněte si místo CLI desktopovou aplikaci z claude.com/download — je to plnohodnotná alternativa.

npm install -g @anthropic-ai/claude-code hlásí permission error

Na macOS a Linuxu nepoužívejte sudo — způsobuje jiné problémy, které se špatně ladí. Místo toho použijte nativní instalátor, který nepotřebuje npm ani Node.js:

curl -fsSL https://claude.ai/install.sh | bash

Vraťte se ke Kroku 2 a zvolte tabs „Terminál → macOS / Linux".

git clone hlásí „repository not found" nebo „permission denied"

Nejspíš jste zkopírovali URL s překlepem nebo se pokoušíte o SSH klon bez klíče. Použijte přesně tuhle HTTPS verzi:

git clone https://github.com/aibilitycz/claude-code-90-webinar.git

Žádné git@github.com:…, žádné ssh://. HTTPS nepotřebuje žádnou autentizaci, protože repozitář je veřejný.
